# Development Workflow

This project will use a simple and clean development workflow based on Git and GitHub.

The `main` branch will hold stable work, while future features can be developed in separate branches when needed. Changes should be committed with clear messages that describe what was added or improved.

Before committing, files should be checked to make sure they run correctly, important documentation is updated, and local secrets such as `.env` are not included in Git. This workflow will help keep the project organized and easy to understand.
